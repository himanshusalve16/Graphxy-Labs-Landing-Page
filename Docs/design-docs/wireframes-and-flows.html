Product CTA Routing Flow (P0 Rule)

To ensure high product engagement and clear education, all product listing cards and references must first route the user to the dedicated product detail page rather than forcing a direct waitlist sign-up popup.

1
User clicks "Explore [Product]" CTA button on Home, Products, or Ecosystem tree cards.
2
Router intercepts path and redirects user to Product Detail Page (/graphzy, /clampbox, /forkline, or /lattice).
3
User views product details and sees the Interactive Preview Module (Boot Attest for Clampbox, Floor alerts for Forkline, Committed funds for Lattice).
4
If interested, the user fills the Waitlist / Inquiry Form directly inline at the bottom of the page.

Responsive Breakpoint Mapping

The system dynamically loads distinct UI graph engines depending on breakpoints and input type to prevent SVG layout coordinates distortion:

M
Mobile Layout (< 640px): Loads `MobileTreeGraph` and `RadialGraph` (scaled viewBox, tap gestures, bottom details overlay sheet to conserve screen width).
T
Tablet Layout (640px - 1024px): Loads `SvgTree` and `RadialGraph` (increased hit ranges, touch/click selection highlights, bottom details panels).
D
Desktop Layout (>= 1024px): Loads standard `SvgTree` and `ScatterGraph` (hover triggers connection pulses, tooltip overlay follows mouse cursor).
