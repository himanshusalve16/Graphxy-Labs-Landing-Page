Motion Principles

Motion signals state change — it never decorates. The result reveal is the signature moment. Everything else is fast, subordinate, and purposeful.

MomentEasingDurationDetail
Result reveal — text + canvasease-out250ms + 80ms staggerFade + 12px rise. Canvas arrives just after text.
Input → breadcrumb collapseease-out280msShared-element. No hard cut.
Section reveal on scrollease-out300msEntire section block. One reveal zone visible at a time.
Product card hoverease-out200mstranslateY(−2px) + shadow deepening. No scale.
Canvas highlight (new expression)ease-in-out500ms ring pulse1px subject-accent ring. One pulse, not looping.
Dashboard countersease-out600ms count-upOne-time, on first dashboard load only.
Button / link hoverease-out120mstranslateY(−1px) only.
Follow-up appendease-out200msFade + rise. Auto-scroll to reveal.

Rules: Settle, don't bounce. Ease-out for entrances; ease-in-out for state changes. No spring, no overshoot. One orchestrated entrance per screen — all else is subordinate and <200ms. Animate section blocks as units, not individual elements within. Respect prefers-reduced-motion: all transitions degrade to opacity-only fades.

13

Accessibility

Accessibility is not an add-on. It's part of the system. Every rule here is a quality signal, not a compliance checkbox.

Color contrast — WCAG AA minimum
All text on colored backgrounds: 4.5:1 for normal text, 3:1 for large text. Never rely on color alone to convey meaning — always pair with a label, icon, or pattern. Warn amber is not used as text on white without a background tint.
Keyboard navigation & focus rings
All interactive elements have a visible focus ring when keyboard-focused: 2px outline, 3px offset, in the corresponding brand or subject accent color. Never remove outline without providing a visible alternative. Tab order follows visual reading order.
Reduced motion
All entrance and transition animations respect prefers-reduced-motion: reduce. Entrance animations (fade + rise) degrade to opacity-only. Pulse animations (thinking state, canvas ring) become static. Slider animation is unaffected — it is direct interaction feedback.
Semantic HTML & ARIA
Use semantic elements: <nav>, <main>, <article>, <button>. Never use <div> with click handlers for interactive elements. All images need descriptive alt text. The Desmos canvas needs an aria-label describing the current graph. Follow-up counter is announced via aria-live="polite".
Touch targets
All interactive elements on mobile: minimum 44×44px touch target (WCAG 2.5.5). Small visual elements (delete icons, tag dots) use padding or ::after pseudo-elements to meet the minimum, even if the visual size is smaller.
Error communication
Errors are never communicated by color alone. All error states use an icon, inline label, and neutral amber tone — never red as a dominant color. Error messages are associated to the field via aria-describedby. Inline banners, not modals or toasts.
